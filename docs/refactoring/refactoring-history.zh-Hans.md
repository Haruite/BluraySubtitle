## 用途

本文档长期记录 Python 重构情况，包括每个已完成阶段或工作流修改了什么、业务逻辑如何变化、执行了哪些测试，以及哪些内容被明确推迟。

Phase 1 的详细分析仍保存在 [第一阶段重构契约与配置矩阵](phase-1-contract-and-configuration-matrix.md) 中。后续所有修改还必须遵守强制性的[代码修改规范](../development/code-standards.zh-Hans.md)。

## 维护规则

每次完成重构或重大改动，都必须在同一次修改中同步更新本文档及其英文版本；一般修改不需要增加历史记录。每条记录必须包含：

- 范围与提交；
- 删除的冗余或冲突路径；
- 业务逻辑变化，包括旧行为和新行为；
- 文档与 i18n 变化；
- 自动检查及仍需使用真实素材完成的手动检查；
- 明确推迟的工作。

历史记录必须遵循作者已经记录的意图；尚未确定的行为应标记为待确认，不能为了补全文档而自行给出结论。

## Phase 2——显式 GUI 与运行时配置

日期：2026-07-20  
提交：`ceb2927`（`refactor: unify GUI and runtime task configuration`）

### 范围

建立从当前 GUI 状态经过 Worker 到 Remux/Encode Service 的唯一显式配置路径。

### 删除的冗余或冲突路径

- 删除旧的全局 `CONFIGURATION` 状态及其导入。
- 删除当前 GUI 配置生成失败后回退到旧快照的逻辑。
- 删除 Worker 在已经显式传参时再次给 `BluraySubtitle.configuration` 赋值的逻辑。
- 删除可能使界面行与运行时值错位的输出名称和行对齐回退。

### 逻辑变化

- 剧集模式在启动时根据当前控件重新生成配置；电影模式在捕获前刷新配置。
- 当前配置为空或无效时停止任务，不再复用旧状态。
- Worker 将捕获到的配置原样传到 Service 边界。
- 分集 Service 必须接收显式配置，并在写入前拒绝无效章节区间。
- 每个已选择的主 MPLS 必须且只能对应一条非空 Remux 命令；缺少或多出命令都会在启动时报错。
- 界面中可见的分集输出名称和行顺序与请求保持一致。
- 可预知的缺失源文件和无效已选行尽可能在执行前报告。

### 文档与 i18n

- 同步更新中英文 README，说明所见即所得和禁止旧配置回退。
- 记录一个主 MPLS 对应一条命令的规则。
- 增加配置和行校验所需的中英文错误信息。

### 验证

- 新增 GUI 配置、命令数量、输出名称、无效区间和 Service 边界测试。
- 新增 RemuxWorker、EncodeWorker 的显式配置传递测试。
- 新增源码完整性测试，确认生产代码中不再存在全局 `CONFIGURATION`。
- 执行 Python 编译/导入、i18n、split 契约和仓库测试。

### 推迟内容

## Phase 3.1——合并字幕工作流

日期：2026-07-20  
提交：`7def4df`（`refactor: unify subtitle merge workflow`）

### 范围

将“合并字幕”重构为剧集和电影模式共用的独立 Request、Worker 和 Service 工作流。

### 删除的冗余或冲突路径

- 删除剧集和电影模式分别重建重叠状态的执行分支。
- 删除旧字幕勾选状态复用；启动时直接读取当前表格已选择行。
- 删除 Worker 对 Service 配置的隐藏预赋值。
- 删除在所有可预知冲突尚未确定前就开始部分写入的路径。

### 逻辑变化

- 一个不可变请求捕获原盘路径、所选字幕、所选主播放列表、后缀、补全目录选项和电影映射。
- 剧集和电影模式使用同一个 Service 入口。
- 支持 SRT、ASS、SSA 和 SUP；同一个合并输出不能混用格式。
- 后缀完全按照 GUI 当前显示值应用。
- 写入前计算所有原盘目录级和播放列表级输出；重复路径或已存在文件会中止任务，绝不覆盖。
- SUP 可通过统一字幕领域接口追加并写出。
- “补全蓝光目录”在剧集和电影模式中保持一致。

### 文档与 i18n

- 同步更新中英文 README，记录支持格式、后缀、输出位置和冲突处理。
- 新增对应的中英文校验及输出信息。

### 验证

- 新增 SRT/SUP 输出、剧集/电影请求捕获、映射错误、格式混用、后缀及已有输出安全测试。
- 新增 MergeWorker 边界测试。
- 执行 Python 编译/导入、i18n、split 契约和仓库测试。

### 推迟内容

本次只修改合并字幕，没有重新设计添加章节、Remux、Encode、SP、音频转换或 Dolby Vision。

## Phase 3.2——添加章节工作流

日期：2026-07-20  
提交：`107cea1`（`refactor: unify add chapters workflow`）

### 范围

将“添加章节”重构为独立的 GUI Request、后台 Worker 和纯数据 Service 路径。

### 删除的冗余或冲突路径

- 删除 GUI 线程中的同步执行。
- 删除吞掉 GUI 配置错误后回退到旧表格/配置逻辑的路径。
- 删除 Service 边界中的 Qt 表格对象。
- 删除对工作目录共享 `chapter.txt` 的依赖。
- 删除对 `completion()` 的调用，避免把“直接编辑原文件”误解为允许补全蓝光目录。

### 逻辑变化

- 请求在启动时一次性捕获当前可见 MKV 顺序、所选主播放列表、准确的源/输出路径对和直接编辑模式。
- MKV 与所选播放列表根据时长和章节点按顺序匹配，不要求文件名包含 `BD_Vol_NNN`。
- 修改任何 MKV 前先完成全部章节文档规划；所选播放列表无法覆盖全部 MKV 时，在写入前失败。
- 主播放列表、MKV 输入、所需 MKVToolNix 程序、重复路径和已有输出在可预知时于 Worker 启动前检查。
- 已存在输出明确报错。
- 每个任务使用独立临时章节文件。
- `mkvmerge`、`mkvpropedit` 不再通过 shell 命令字符串执行，失败返回码会被报告；MKVToolNix 返回码 1 仍视为带警告的成功。
- 只有单个零点章节时，直接编辑仍保持源文件不变；新文件模式仍会生成 MKV，但不会加入这个无意义章节。

### 文档与 i18n

- 同步更新中英文 README，记录顺序、播放列表匹配、直接编辑/新文件模式、预检查和输出冲突规则。
- 新增工作流、工具、映射和命令失败所需的中英文信息。
- 同步 Worker 导出以及 GUI/Service IDE 基类契约。

### 验证

- 新增不可变请求捕获、直接编辑、当前表格顺序、输出冲突、无卷号文件名的多播放列表顺序匹配、写入前映射失败、Worker/Service 边界及单章节输出测试。
- 集中执行仓库测试，共 49 项全部通过。
- Python 编译、i18n 审计、split 契约审计、`git diff --check` 和 CRLF 检查通过。

### 仍需真实素材验证

- 使用真实 MKV 和 MKVToolNix 测试新文件模式。
- 使用可丢弃的 MKV 副本测试直接编辑模式。
- 已有输出启动前报错。
- 表格可见顺序调整。
- 可选的多主 MPLS 顺序匹配。

### 推迟内容

Remux/Encode 内部仍在使用的旧章节入口暂时保留，等待对应工作流重构；本次未修改它们的整体调度。

## Phase 3.3——原盘 Remux 工作流

日期：2026-07-22
提交：包含在本次修改中

### 范围

重构 Remux 的 GUI 请求、Worker 边界、预执行规划、主播放列表执行和最终输出映射。保留已有 fallback 算法，但每个已选择主播放列表现在都在一个确定的独立任务中执行。

### 删除的冗余或冲突路径

- 删除 Worker 中镜像 GUI 状态的大量参数和重复的 Service 配置赋值。
- 删除按原盘分组后执行的路径；旧路径在同一原盘存在多个已选择主播放列表时可能只执行第一个。
- 删除提前创建输出目录、随后再扫描目录以重新发现本任务输出的逻辑。
- 删除输出名称字符自动替换、数字冲突后缀和回退到无关原始输出文件的逻辑。
- 删除强制启用“补全蓝光目录”的逻辑。

### 逻辑变化

- 一个自有 Remux 请求一次性捕获当前 GUI 配置、已选择主播放列表、可见输出名称和语言、SP 行、轨道设置、默认音频编码、电影模式、裁剪、Dolby Vision 和目录补全选项。
- 按界面可见顺序规划主播放列表。每个已选择主播放列表必须生成且只生成一条非空命令和一个执行任务，同一原盘中的多个主播放列表也不例外。
- GUI 命令预览与执行逻辑现在共同使用同一个七项命令结果。命令生成失败时预览保持为空，并在捕获配置时被拒绝，不再合成一条缺少计划输出路径和轨道参数、但看起来可以执行的命令。
- 创建输出目录前推导所有命令理论输出和最终 GUI 输出路径，并在写入前校验配置行数、章节范围、重复路径和已存在文件。
- 最终名称严格按照界面显示值应用；没有 `.mkv` 扩展名时补充扩展名，无效的 Windows 文件名明确报错。
- 主命令或 fallback 必须生成对应任务规划的全部输出。缺少输出时任务现在会失败，不再静默跳过；失败后删除本任务新建的预期中间输出。
- 收尾处理直接使用规划输出列表，不再扫描目标目录，并使用任务独立的临时章节文件。
- 无论是否勾选目录补全，章节元数据都会直接写入本任务新生成的输出；该复选框现在只控制蓝光目录补全。
- 每个主播放列表任务都会捕获“编辑轨道”保存的语言，并在创建输出前检查 `mkvpropedit` 是否可用。主命令或任一 fallback 成功后，只对实际包含轨道应用已配置语言，修改后重新识别输出并验证；映射、命令或验证失败都会使任务失败并删除该任务新建的主输出。
- 剧集输出顺序继续与配置顺序对齐，供后续字幕、语言、SP、音频和 Dolby Vision 处理使用。
- “补全蓝光目录”严格使用捕获到的复选框值。

### 文档与 i18n

- 同步更新中英文 README，记录一个主播放列表对应一个任务、写入前输出规划、精确命名、冲突处理、严格完整性、复选框所有权和经过验证的主输出语言修正。
- 新增校验、命令失败、缺少输出、输出映射和语言修正所需的中英文消息。
- 更新 Service IDE 兼容声明和仓库内批量电影调用方。
